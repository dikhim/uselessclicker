Version 0.5.0

11.06.2019 Interface changes, similar image searching, optimizations, and bugfixing.

Version 0.4.1

User interface

  • Now all inserted text inserts with intents
  • Select text and press tub to insert intents to each selected line
  • Context menu item 'Insert'

Other

  • Language selection window for the first launch
  • Added "Insert file path" button in the recording section
  • Load an image from selected text. If you have a selection it at first tries to load image from selection and if an attempt failed than showed an input dialog. To load image you should select any text between (' and '), like "ate.image('H4sIAAAAAAAAAOsM8HPn5ZLiYmBg4PX0cAkC0ixAzMjBBiR/2vCfBVLCni6OIRW3knfUvP5YXytcbcC2kTHUkitRMzM'+ '6FCjN4Onq57LOKaEJAJPFrSFMAAAA');" is still correct. It omits any unnecessary chars between braces


Version 0.4.0

Screen capturing

  • Added the screen object, with which you can capture an image from the display;
  • added the Screen tab, where you can quickly select the part of the screen and save it.

Images

  • a new Image class that contains all the necessary methods for image processing;
  • searching for an image in another image using the findFirst, findAll, and findLimit methods;
  • images can be compiled by calling the method image.compile(), thereby consuming time for the compilation, but speed up further search;
  • added the image output view, where an image can be loaded, edited, saved to file or inserted to a script;
  • you can upload an image to the image output view from the script using the system.showImage() method.

Keyboard

  • Added new method key.typeText(String layout, String text) that typing the passed text in the specified layout;
  • the method key.typeText() supported layouts:
    "us" - English(US),
    "gb" - English(UK),
    "ru" - Russian,
    "ua" - Ukrainian.
  • Added the key ALT_GR for testing

Other

  • Added new object create by which it is easy to create necessary objects
