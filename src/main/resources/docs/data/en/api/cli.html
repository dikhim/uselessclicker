Command line parameters

  • -h,--help -> show help
  • -g,--gui >enable> -> if false, then run cli application. The default value is true. It helps if there is no need to start a gui. Saves RAM an CPU usage
  • -H,--http-server >port> -> set a port for the HTTP server and start it
  • -S,--socket-server >port> -> set a port for the Socket server and start it
  • -o,--open-file >file path> -> open file in the script editor
  • -r,--run-file >file path> -> open file in the script editor and run it
  • -R,--recording >enable> -> if false, then mouse and keyboard recording will be disabled. The default value is true. Kills the mouse and keyboard capturing thread to save some RAM and CPU usage. No mouse/keyboard recording, no onKey/onMouse/onShortcut method calls, no stop/run script shortcuts, etc.

Examples

For example, we have an application file Useless_Auto_Clicker.jar and some scripts in the folder /home/user/scripts/


Open a certain script:

java -jar Useless_Auto_Clicker.jar -o=/home/user/scripts/script1.js


Run a certain script:

java -jar Useless_Auto_Clicker.jar -o=/home/user/scripts/script1.js


Run a certain script without gui:

java -jar Useless_Auto_Clicker.jar -g=false -o=/home/user/scripts/script1.js


Run a Socket server without gui and mouse/keyboard recording:

java -jar Useless_Auto_Clicker.jar -g=false -R=false -S=5000
